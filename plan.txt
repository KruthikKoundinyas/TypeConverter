PHASE 1 — Project Setup (Prompt)
I want to build a fully client-side file converter web app (no backend at all).

Requirements:
- Pure frontend (HTML, CSS, JavaScript)
- No server or API calls
- Clean folder structure
- Ready for deployment on Vercel or Netlify

Set up:
- index.html
- styles.css
- main.js

Include:
- Drag and drop file upload UI
- File input fallback
- Basic layout (header, upload area, result area)

Keep the code simple and modular.
Do not include any backend code.
PHASE 2 — ffmpeg.wasm Integration (Prompt)
Add client-side audio conversion using ffmpeg.wasm.

Requirements:
- Use @ffmpeg/ffmpeg
- Load ffmpeg only when needed (lazy loading)
- Show "Loading engine..." while initializing
- Convert one audio file (e.g., WAV to MP3)

Include:
- Proper async handling
- Error handling
- Clear UI feedback

Keep it beginner-friendly but production-quality.
PHASE 3 — Image Conversion (Prompt)
Add image conversion using the Canvas API.

Requirements:
- Convert images (PNG, JPG, WEBP)
- Use canvas.toBlob for conversion
- Support quality adjustment (compression)
- Show preview before and after

Do not use any backend or Node.js libraries.
Everything must run in the browser.
PHASE 4 — Batch Processing (Prompt)
Extend the app to support batch file conversion.

Requirements:
- Allow multiple file uploads
- Process files one by one (queue system, not parallel)
- Show progress:
  - Current file name
  - Number of files processed (e.g., 2/5)
- Prevent UI freezing

Keep the code clean and modular.
PHASE 5 — Compression Feature (Prompt)
Add file compression controls.

Requirements:
- Add a quality slider (10–100)
- Apply compression:
  - Images: adjust quality via canvas
  - Audio: adjust bitrate in ffmpeg (64k, 128k, 320k)
- Show file size before and after conversion

Focus on good UX and clear feedback.
PHASE 6 — Conversion History (Prompt)
Add a conversion history feature using localStorage.

Requirements:
- Store:
  - file name
  - input format
  - output format
  - size before and after
  - timestamp
- Display history in a list
- Add "Clear history" button
- Persist data across page reloads

Do not use any backend.
PHASE 7 — UI Polish (Prompt)
Improve the UI/UX of the file converter.

Requirements:
- Modern clean design
- Drag-and-drop highlight effect
- Loading spinner during conversion
- Disabled buttons while processing
- Responsive design (mobile-friendly)
- Light/Dark mode toggle

Focus on usability and visual clarity.
PHASE 8 — Performance Optimization (Prompt)
Optimize the app for performance.

Requirements:
- Lazy load ffmpeg.wasm
- Avoid memory leaks (revoke object URLs)
- Handle large files gracefully
- Add basic error recovery

Explain what each optimization does.
PHASE 9 — Final Review (Prompt)
Review my entire project and suggest improvements.

Focus on:
- Code structure
- Performance
- UX improvements
- Edge cases
- Best practices

Be critical and specific.
Pro Tips (this is what actually makes Claude useful)
Don’t just dump everything at once. Do this instead:
Good workflow:
Give ONE phase prompt
Test code
Come back and say:
This part is working. Now extend it to...
What to avoid:
“Build full app with all features” → messy output
Copy-pasting without understanding
Not testing between steps
Final advice
If you follow this properly, you’ll end up with:
A clean, fully client-side WebAssembly-powered app
—not a chaotic AI-generated mess.
